Lets check out some code 

#cssrules {
    font-size: 20pt;
}
div.content {
    font-size: 10pt;
    background-color: #bada55;
    margin: 20px;
}
div.content pre {
    margin: 10px;
    text-shadow: 2px 2px 2px #333;
    border-radius 5px;
}

That's a lot of double maintenance code…

How about we add some variables?

$bgcolor: #bada55;
$fsize: 20pt;
 
#cssrules {
    font-size: $fsize;
}
div.content {
    font-size: 10pt;
    background-color: $bgcolor;
    margin: 20px;
}
div.content pre {
    margin: 10px;
    text-shadow: 2px 2px 2px #333;
    border-radius 5px;
}

Let's put in some arithmetic

$bgcolor: #bada55;
$fsize: 20pt;
$common_margin: 10px;
 
#cssrules {
    font-size: $fsize;
}
div.content {
    font-size: $fsize / 2;
    background-color: $bgcolor;
    margin: $common_margin * 2;
}
div.content pre {
    margin: $common_margin;
    text-shadow: 2px 2px 2px #333;
    border-radius 5px;
}

How about some lovely nesting? 

$bgcolor: #bada55;
$fsize: 20pt;
$common_margin: 10px;
 
#cssrules {
    font-size: $fsize;
}
div.content {
    font-size: $fsize / 2;
    background-color: $bgcolor;
    margin: $common_margin * 2;
    pre { 
        margin: $common_margin;
        text-shadow: 2px 2px 2px #333;
        border-radius 5px;
    }
}

Can we do something about prefixes for border-radius?
Something like a mixin? 

@mixin common-border {
  $b: 5px;
  -webkit-border-radius: $b;
  -moz-border-radius: $b;
  -o-border-radius: $b;
  border-radius: $b;
}

Usage:

@include common-border;
// variables and mixin defs...
 
#cssrules {
    font-size: $fsize;
}
div.content {
    font-size: $fsize / 2;
    background-color: $bgcolor;
    margin: $common_margin * 2;
    pre { 
        margin: $common_margin;
        text-shadow: 2px 2px 2px #333;
        @include common-border;
    }
}
